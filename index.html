    About

    About this build — architecture & implementation notes. This system is a fully static, zero-build, dependency-free single-page application (vanilla ES6, IIFE-module pattern per file, no bundler/transpiler/framework runtime) engineered for stateless edge delivery via GitHub Pages, with client-side persistence handled entirely through the browser's localStorage API (profile state, quiz-history, planner-completion sets, and a crypto.randomUUID()-derived session identifier used for server-side conversational memory continuity). The conversational layer is backed by an n8n Cloud–hosted orchestration workflow exposing a CORS-enabled HTTPS webhook trigger; the workflow composes a Retrieval-Augmented Generation (RAG)–style AI Agent node with tool-calling access to a structured knowledge base (course catalog, syllabi component-weight breakdowns, professor metadata), a live web-search tool for dynamic queries (foreign admissions, tuition, exchange programs), and a dedicated data-table-backed retrieval tool (lecture_content) populated by a parallel ingestion pipeline: a staff-facing n8n form accepts uploaded .pptx/.pdf lecture materials, extracts raw text (zip/XML unpacking for OOXML slide decks, native PDF text extraction), and pipes it through an LLM-based structuring pass (Claude) that normalizes unstructured slide content into typed per-lecture records (topic, subtopics, key points, quiz-generation hints) — meaning quiz generation is materially grounded in professor-authored source content rather than parametric model knowledge alone, with graceful degradation to general-domain reasoning (explicitly disclosed to the end user) when no grounding material exists for a given course/week pair. The request/response contract between client and workflow is a fixed JSON envelope discriminated on a `type` field (text | quiz | resources), and the client implements defensive, non-brittle parsing: a cascading extraction strategy (direct JSON.parse → fenced-code-block extraction via regex → brace-depth-tracked balanced-object scanning with string-literal and escape-sequence awareness) recovers valid envelopes even when the upstream LLM prepends natural-language meta-commentary or wraps output in markdown code fences, ensuring high structural fault-tolerance against non-deterministic LLM output formatting. Client-side analytical features are implemented as deterministic, explainable rule-based systems rather than opaque ML models: the Strength Analyzer computes a coverage-weighted dot-product similarity score between a student's per-category GPA vector (derived via credit-hour-weighted averaging across a discrete letter-grade-to-4.0-scale mapping) and each specialization field's hand-tuned weight vector, applying a coverage-shrinkage penalty (0.6 + 0.4 × categoryCoverageRatio) to prevent single-category grade data from producing overconfident matches in multi-dimensional specializations; the Quizzes module filters the course catalog to a materially-grounded subset (only subjects with confirmed lecture_content ingestion) before dispatching a quiz-mode request, then performs client-side answer-set diffing post-grading to construct a targeted remediation payload (per-question chosen-vs-correct deltas) that is round-tripped to the AI Agent for citation-backed resource discovery and focus-area recommendation — closing the loop from assessment to personalized remediation without server-side state beyond the ephemeral session. The entire UI is internationalized to Georgian (ka) with Noto Sans Georgian web-font fallback, includes prefers-reduced-motion and prefers-color-scheme media-query accessibility affordances, WCAG-conscious focus-visible states, ARIA live-region announcements for asynchronous chat updates, and is responsive down to a 360px viewport floor with zero horizontal-overflow guarantees verified via automated headless-browser (Playwright/Chromium) regression testing across all interactive flows (multi-turn dialogue state, quiz grading/scoring, resource-card rendering, GPA computation, and prerequisite-graph resolution) prior to every deployment.
